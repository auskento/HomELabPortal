## Summary

Comprehensive authentication system refactoring implementing three authentication methods: Basic Auth, Entra ID (Microsoft), and Google OAuth. Includes auth-specific dashboard selection, configuration updates, and Unraid template improvements.

### 2. Authentication Method Features

**Basic Auth**
- Simple HTTP Basic Auth with bcrypt-hashed credentials
- Automatically forces classic interface (STYLE=classic)
- Direct service links dashboard (no iframes, no re-prompting)
- No external dependencies

**Entra ID (Microsoft)**
- Azure AD / Microsoft Entra authentication
- Redirect URI: https://domain/auth/oauth2/callback
- Modern dashboard with embedded iframes
- User info display and logout button
- Session management

**Google OAuth**
- Google account / Workspace authentication  
- Redirect URI: https://domain (domain root only)
- Modern dashboard with embedded iframes
- User info display and logout button
- Session management

## Key Design Decisions

1. **Auth Method Exclusivity**: Only one auth method can be active at a time to prevent conflicts
2. **Dashboard Based on Auth**: Basic auth uses direct links, OAuth methods use iframes with session state
3. **Automatic Interface Enforcement**: Basic auth forces classic interface in entrypoint
4. **Different Redirect URIs**: Entra uses /auth/oauth2/callback, Google uses domain root
5. **Backward Compatibility**: Old OAUTH2_* and OFFICE365_* variables replaced with method-specific ones

## Migration Notes

For existing users upgrading from Office365 OAuth:
1. Replace OAUTH2_* variables with ENTRA_*
2. Replace OIDC_PROVIDER_METADATA_URL with ENTRA_PROVIDER_METADATA_URL  
3. Set AUTHTYPE=entra (instead of relying on flags)
4. Redirect URI format stays the same: /auth/oauth2/callback
